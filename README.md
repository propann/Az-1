# Az-1
Description du DIN MIDI (le “dit”)
Le DIN MIDI, c’est le port de communication des anciens dieux.
Cinq broches. Zéro pitié. Toujours en service depuis les années 80 pendant que l’USB a déjà changé de version plus souvent que ton noyau Linux.

Ici, pas de son.
Le son, c’est vulgaire.
Le MIDI envoie des ordres.

« Joue cette note. Maintenant. Arrête. Merci. »

Comment ça marche (version underground)

Débit sacré : 31 250 bauds

Protocole minimaliste : pas d’auth, pas de chiffrement, mais personne n’ose l’attaquer

Chaque bit est une intention musicale pure

Le MIDI ne discute pas.
Il commande.

Pourquoi les hackers aiment le DIN MIDI

Aucun driver exotique

Aucun firmware cloud

Aucun compte à créer

Fonctionne même après l’effondrement numérique

Un câble MIDI survivra à :

trois OS,

deux DAW,

un crash de kernel,

et probablement à l’humanité.

Sécurité niveau bunker

Grâce à l’optocoupleur, le MIDI vit dans son monde :

pas de masse partagée,

pas de boucle de terre,

pas de court-circuit existentiel.

Ton Teensy peut paniquer,
le synthé reste zen.

Dans le projet AZ-1

Le DIN MIDI est le rituel d’invocation.
C’est lui qui permet à ton firmware moderne de murmurer à des machines mythiques sans les réveiller brutalement.

Une résistance.
Une diode.
Un opto.

Et la musique obéit.
